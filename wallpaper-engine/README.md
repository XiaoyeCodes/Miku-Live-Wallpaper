# 时光锁屏场景（Wallpaper Engine 网页壁纸）

## 已包含的行为

- 根据电脑本地时间自动选择场景。
- 相邻场景在两个时间锚点之间持续交叉淡化；淡化曲线使用 `smootherstep`，起止都不会突变。
- 仅显示当前与下一张高分辨率图片，并提前请求再下一张，兼顾衔接与显存占用。
- 使用 `object-fit: cover`，可适配 16:9、超宽屏及其他显示器比例。
- 音乐播放时显示右上角的节奏光环与底部频闪条；低频鼓点会驱动光环大小和亮度。

## 时间锚点

| 图片 | 最接近完整呈现的时间 |
| --- | --- |
| 晨曦 | 05:00 |
| 早晨 | 07:00 |
| 上午 | 09:00 |
| 中午 | 12:00 |
| 下午 | 14:00 |
| 傍晚 | 17:30 |
| 晚上 | 20:00 |

例如，早晨场景会从 07:00 开始逐渐混合至上午场景，并在 09:00 时完全成为上午场景。晚上则自然过渡到次日 05:00 的晨曦。

## 导入 Wallpaper Engine

1. 打开 Wallpaper Engine，点击 **创建壁纸**。
2. 将本文件夹中的 `index.html` 拖入创建界面（或选择网页壁纸并选中它）。
3. 在编辑器预览不同时间段；满意后通过 **Workshop → Share Wallpaper** 发布。

首次导入时 Wallpaper Engine 会在项目目录生成 `project.json`。不要把桌面上的原压缩包放到导入目录中，以免它被一并打包。

音乐可视化依赖 Wallpaper Engine 的系统音频接口；首次导入后请在编辑器中保存一次项目，确保自动生成的 `project.json` 启用音频处理。普通浏览器预览或系统静音时，这层效果会保持透明。

## 华硕 Aura Sync 机箱灯效（可选）

项目已包含 `aura-sync.js`：当 Aura Ready Game SDK 的本机 REST 服务可用时，它会在检测到音乐后自动接管 Aura 设备，并按约 80 ms 的间隔以青蓝到粉色的鼓点脉冲刷新所有已枚举设备。音乐暂停超过约 1.5 秒、壁纸关闭或切换时，会释放控制并恢复 Armoury Crate/Aura 原来的效果。

此功能要求：

1. 主板/ARGB 控制器及机箱灯带本身支持 ASUS Aura Sync。
2. 安装并运行提供 `http://127.0.0.1:27339/AuraSDK` 服务的 **Aura Ready Game SDK** 兼容组件。
3. 导入壁纸后在 Wallpaper Engine 中播放音乐并保存项目一次。

播放音乐期间若没有检测到该服务，壁纸会每 30 秒重新尝试连接，但不会修改任何灯效。Aura SDK 3.1 的 COM/Lighting Service 接口与此 REST 接口不同；若你的设备仅支持 SDK 3.1，需要另行安装一个 Windows 桥接程序后才能接入。

## 调整时段

编辑 `wallpaper.js` 顶部的 `scenes` 数组中每一项的 `minute`：例如 `17 * 60 + 30` 表示 17:30。图片路径和顺序也可在同一处修改。
